## 2. システム構成図（Django）

```mermaid

    end

    Django --> PG
```

**MVP では Redis / Celery / メールサービスは導入しない**。画像リサイズは Django プロセス内で同期実行（Pillow）。

## 4. スケジュール・通知のシーケンス図（**Phase 3** で実装）

> MVP では通知は実装しない。本シーケンスは Phase 3 の参照設計。

```mermaid
sequenceDiagram
    participant Beat as Celery Beat
    participant Task as 通知タスク
    participant DB as PostgreSQL
    participant Mail as Resend
    Beat->>Task: 定期実行（例: 5分毎）
    Task->>DB: next_run_at が通知タイミングのスケジュール取得
    Task->>DB: ユーザーの notify_start_time / notify_end_time を確認
    Task->>Mail: メール送信（ディープリンク付き）
    Task->>DB: next_run_at を次回日時に更新
```

## 5. 「今月のやること」生成シーケンス（P1, P2 中心）

[開発前検討事項.md](./開発前検討事項.md) §10.1 の決定により、**A（オンデマンド計算）+ 完了状態のみ別テーブル `care_schedule_completions`** に保存する方式。サービス層 `apps/schedules/services/todos.py::compose_monthly_todos()` が合成を担う。

```mermaid
sequenceDiagram
    participant User as ユーザー
    participant Web as Django View
    participant Svc as todos service
    participant DB as PostgreSQL
    User->>Web: GET /home/ (ホーム表示)
    Web->>Svc: compose_monthly_todos(user, year_month)
    Svc->>DB: care_schedules: 当月の next_run_at（②ユーザー個別）
    Svc->>DB: bonsai_species.monthly_tasks: 所有品種の今月推奨作業（①品種マスタ由来）
    Svc->>DB: monthly_advices: 当月の共通アドバイス（③共通）
    Svc->>DB: care_schedule_completions: 当月の完了状態を LEFT JOIN
    Svc-->>Web: ToDo一覧（合成済み + 完了状態付き）
    Web-->>User: 「今月のやること」HTML フラグメント（HTMX）
    User->>Web: ToDoタップ
    Web-->>User: ログ記録画面（プリフィル済み）へ遷移
    User->>Web: ログ保存（POST）
    Web->>DB: care_logs INSERT
    Web->>DB: care_schedule_completions INSERT（status=done, log_id=...）
```

## 6. 通知駆動UX シーケンス（P6 タカシ）— **Phase 3** で実装

> MVP では通知機能を提供しないため、本シーケンスは Phase 3 の参照設計。

```mermaid
sequenceDiagram
    participant Beat as Celery Beat
    participant Mail as Resend
    participant User as ユーザー
    participant Web as Django View
    participant DB as PostgreSQL
    Beat->>Mail: 通知メール送信（ToDoのディープリンク付き）
    Mail-->>User: メール受信
    User->>Web: ディープリンクタップ /todos/{id}/
    Web->>DB: 対象ToDo取得 → ログ記録画面プリフィル
    Web-->>User: プリフィル済みフォーム（対象盆栽・作業種別・日時）
    User->>Web: 「保存」タップ（ワンタップ完了）
    Web->>DB: care_logs INSERT
    Web->>DB: care_schedule_completions INSERT（status=done, log_id=...）
```

## 7. 一括ログ記録シーケンス（P4 ソウタ）— **Phase 3** で実装

```mermaid
sequenceDiagram
    participant User as 専門家
    participant Web as Django View
    participant DB as PostgreSQL
    User->>Web: 盆栽一覧 → 品種別グルーピング表示
    User->>Web: 真柏10鉢を一括選択
    User->>Web: 「一括ログ記録」起動
    Web-->>User: 一括フォーム表示（作業種別・日時・天候）
    User->>Web: フォーム入力 → 「全鉢に記録」
    Web->>DB: care_logs INSERT × N（トランザクション内）
    Web-->>User: 完了サマリー（10件記録）
```

# 盆栽管理アプリ ユーザーペルソナ定義

本アプリの設計判断・優先度づけ・UIライティングのよりどころとなるペルソナ定義です。
過去のサイトマップ評価で言及した6種に加え、運営側ペルソナ（管理者）を含めた **計7ペルソナ** を整理します。

---

## ペルソナ全体マップ

| 区分 | # | 名称 | 利用頻度 | 鉢数 | 重要度 |
|------|---|-----|---------|------|--------|
| エンドユーザー | P1 | 入門者ハル | 週2〜3回 | 1〜2鉢 | ★★★ |
| エンドユーザー | P2 | 学習中級者ミドリ | ほぼ毎日 | 5〜15鉢 | ★★★ |
| エンドユーザー | P3 | 多忙ライフスタイル ケイタ | 不定期 | 3〜6鉢 | ★★ |
| エンドユーザー | P4 | 多鉢運用専門家 ソウタ | 毎日 | 30鉢以上 | ★★ |
| エンドユーザー | P5 | 鑑賞・記録派 サキ | 週1〜2回 | 2〜5鉢 | ★★ |
| エンドユーザー | P6 | 通知駆動・受動的利用者 タカシ | 通知契機のみ | 3〜10鉢 | ★★ |
| 運営 | P7 | コンテンツ管理者 ナナ | 週3〜5回 | ─ | ★★★ |

★★★ = MVP段階で必ず満たすべきペルソナ
★★   = MVP後 / Phase 2 で重視するペルソナ

## P1: 入門者ハル（盆栽を始めたばかり）

### プロフィール
- **氏名（仮）**: 春田 ハル（Hal Haruta）
- **年齢**: 28歳
- **性別**: 女性
- **居住**: 東京都・マンション住まい（ベランダあり）
- **職業**: 会社員（広告代理店、デザイナー職）
- **デジタルリテラシー**: 高い（普段からアプリ利用に慣れている）
- **盆栽歴**: 2ヶ月（誕生日プレゼントで黒松の小品を1鉢もらった）
- **所有鉢数**: 1〜2鉢
- **品種**: 黒松（プレゼント）

### 動機・コンテキスト
- せっかく頂いたものを枯らしたくない
- 「盆栽は難しそう」というイメージがあり、調べる手間を減らしたい
- インテリアとしての美しさにも惹かれている
- 仕事が忙しいので、効率よくケアしたい

### 課題（ペインポイント）
1. **何を、いつすればいいか分からない**（最大の課題）
2. 用語が難しい（剪定／芽摘み／針金かけの違いが不明）
3. 水やりの頻度・量が判断できない
4. 季節ごとの作業が品種で違うことを知らない
5. 失敗が怖くて作業に踏み出せない

### ゴール
- 「アプリを開けば今月やるべきことが分かる」
- 「自分の黒松を1年枯らさず維持できる」
- 「徐々に盆栽の知識が身につく」

### 利用シーン
- 朝の通勤前、ベランダで水やり前にアプリを確認
- 週末、お役立ち記事をじっくり読む
- 作業後、写真とメモを記録（SNS的な楽しみ方）

### 重要機能
- 🟢 ホーム「今月のやること」（最重要）
- 🟢 品種推奨スケジュール自動生成
- 🟢 品種マスタ詳細（黒松の年間ケア）
- 🟢 お役立ち記事（用語解説・基礎知識）
- 🟡 通知（やり忘れ防止）

### 不要機能
- 一括操作・統計・専門用語の細分化

### 行動原則（Quote）
> 「難しい用語より、写真と『今月これだけやれば大丈夫』が知りたい」

## P2: 学習中級者ミドリ（複数鉢を意識的に管理）

### プロフィール
- **氏名（仮）**: 緑川 ミドリ（Midori Midorikawa）
- **年齢**: 42歳
- **性別**: 男性
- **居住**: 神奈川県・戸建て（庭・棚場あり）
- **職業**: メーカー研究職
- **デジタルリテラシー**: 中〜高
- **盆栽歴**: 3年
- **所有鉢数**: 5〜15鉢
- **品種**: 黒松、真柏、楓、五葉松、ケヤキ、寄せ植え1セット

### 動機・コンテキスト
- 趣味として腰を据えて取り組んでいる
- 品種ごとに作業時期が違うことを理解しているが、覚えきれない
- 真柏の芽摘み、黒松の古葉抜きなど**品種固有の作業**を逃したくない
- 紙の手帳・スプレッドシートで管理しているが煩雑になってきた

### 課題（ペインポイント）
1. **品種ごとの特殊作業の時期管理が複雑**（最大の課題）
2. 鉢数が増えると「今月どの鉢に何を」の整理が大変
3. 過去の作業記録（剪定強度、施肥量）を後から振り返りたい
4. スプレッドシート管理だと写真と紐付かない
5. 季節の進行を年単位で俯瞰したい

### ゴール
- 「品種ごとの年間カレンダーを参照しながら計画的に作業」
- 「過去ログを蓄積して、来年・再来年の判断材料にする」
- 「鉢数が増えても作業漏れゼロ」

### 利用シーン
- 朝の作業前、月別スケジュールで「今日は真柏の芽摘み」を確認
- 作業後、肥料の N-P-K や強剪定の度合いを詳細に記録
- 週末、年間スケジュールで来月以降の計画を立てる
- 季節の変わり目に、品種マスタ詳細で再学習

### 重要機能
- 🟢 月別スケジュール（盆栽別／品種別フィルタ）（最重要）
- 🟢 年間スケジュール（モバイル版）
- 🟢 品種マスタ詳細
- 🟢 ログ記録 詳細版（5_1〜5_5、肥料 N-P-K）
- 🟢 盆栽別の作業履歴
- 🟡 タグ管理（管理棚別、樹齢別など）

### 不要機能
- ソーシャル共有・初心者向けの過剰な解説

### 行動原則（Quote）
> 「品種ごとに違う作業を、自分の鉢に合わせて1画面でいつでも確認したい」

## P3: 多忙ライフスタイル ケイタ（旅行・出張多い）

### プロフィール
- **氏名（仮）**: 渓田 ケイタ（Keita Kaida）
- **年齢**: 35歳
- **性別**: 男性
- **居住**: 大阪府・マンション
- **職業**: 営業職（出張多め、月1〜2回家を空ける）
- **デジタルリテラシー**: 高い
- **盆栽歴**: 5年
- **所有鉢数**: 3〜6鉢
- **品種**: 黒松、楓、五葉松

### 動機・コンテキスト
- 出張で1週間〜10日家を空ける
- その間、家族（妻・両親）に水やりを依頼することが多い
- 家族は盆栽の知識がないので、何をすべきか伝える必要がある
- 自分も短期で必要作業を確認できれば嬉しい

### 課題（ペインポイント）
1. **不在期間中の作業を家族にどう伝えるか**（最大の課題）
2. 月単位の表示だと「次の10日間」が抽出できない
3. 共有手段がない（口頭やメモではミスが起きる）
4. 帰宅後、不在中の状態を確認したい

### ゴール
- 「期間を指定して必要作業リストを抽出できる」
- 「家族にメール／LINE で送れる形でエクスポートしたい」
- 「帰宅後に状況確認＆過去分のログを自分で記録できる」

### 利用シーン
- 出張前夜、ホームから「今週」「次の10日」で必要作業を抽出
- 共有エクスポート → メール／LINE で家族へ送付
- 帰宅後、家族の口頭報告を元に過去日付でログ記録

### 重要機能
- 🟢 ホームToDo「期間切替」（今月／今週／カスタム）（最重要）
- 🟢 共有エクスポート（PDF/画像/テキスト）
- 🟢 過去日付でのログ記録
- 🟡 通知（不在中も自分にリマインド）

### 不要機能
- 多鉢の専門的な統計、初心者向けガイド

### 行動原則（Quote）
> 「『次の10日にやること』だけ抜き出して、家族に渡せる形にしたい」

## P4: 多鉢運用専門家 ソウタ（30鉢以上）

### プロフィール
- **氏名（仮）**: 蒼田 ソウタ（Sota Aota）
- **年齢**: 58歳
- **性別**: 男性
- **居住**: 埼玉県・戸建て（広い棚場あり）
- **職業**: 退職後（元教員）
- **デジタルリテラシー**: 中（スマホは使えるがPC寄り）
- **盆栽歴**: 25年（地元盆栽会の役員）
- **所有鉢数**: 30〜80鉢
- **品種**: 真柏多数、黒松、五葉松、楓、雑木類、寄せ植え複数

### 動機・コンテキスト
- 趣味の領域を超えた本格運用
- 同品種を複数本所持しており、同じ作業を一括で行うことが多い
- 過去ログを蓄積して、樹勢・成長傾向を分析したい
- 展示会への出品もあり、特定鉢の状態を厳密に追いたい

### 課題（ペインポイント）
1. **30鉢以上を1鉢ずつ記録するのは現実的でない**（最大の課題）
2. 品種別に絞った表示・操作が必要
3. 過去5年分の履歴を後から検索したい
4. 紙の管理台帳から移行できるツールが少ない
5. タグ／棚場分類で運用したい

### ゴール
- 「同品種・同作業を一括で記録」
- 「品種別に分類された一覧で運用」
- 「年単位の成長・施肥傾向を分析」
- 「展示会前に特定鉢の履歴を一覧」

### 利用シーン
- 朝の棚場で「真柏10鉢に一括施肥記録」
- 月初に品種別ビューで「今月の真柏全体の作業」を計画
- 展示会前、特定鉢の作業履歴・写真を時系列で確認
- 年末、過去1年の活動サマリーを振り返り

### 重要機能
- 🟢 一括ログ記録（複数選択 → 同一作業）（最重要）
- 🟢 盆栽一覧の品種別グルーピング表示
- 🟢 タグ管理（棚場別／樹齢別）
- 🟢 年間スケジュール
- 🟡 検索・期間絞込
- 🟡 統計・サマリー表示（Phase 2）

### 不要機能
- 初心者向けガイド・SNS的演出

### 行動原則（Quote）
> 「30鉢を1鉢ずつ操作させるアプリは使い物にならない。一括と分類が命」

- **性別**: 女性
- **居住**: 京都府・町家
- **職業**: フリーライター
- **デジタルリテラシー**: 高い

### 重要機能
- 🟢 メディア（年別ギャラリー _12）（最重要）

- 🟢 作業ログとメディアの紐付け

### 不要機能

### 行動原則（Quote）

## P6: 通知駆動・受動的利用者 タカシ

### プロフィール
- **氏名（仮）**: 高井 タカシ（Takashi Takai）
- **年齢**: 47歳
- **性別**: 男性
- **居住**: 千葉県・戸建て
- **職業**: IT管理職
- **デジタルリテラシー**: 高い
- **盆栽歴**: 4年
- **所有鉢数**: 3〜10鉢
- **品種**: 黒松、真柏、五葉松

### 動機・コンテキスト
- 仕事が忙しく、能動的にアプリを開く時間がない
- メール／プッシュ通知で「やるべきこと」が来たら対応する受動型
- 1タップで完了したい（情報量よりスピード優先）
- 忘れていても通知で取り戻せれば良い

### 課題（ペインポイント）
1. **能動的にアプリを開く動機がない**（最大の課題）
2. 通知を見てもアプリを開くのが面倒
3. 詳細記録は時間がかかってやらなくなる
4. 通知のタイミングが朝早すぎたり、夜遅すぎたりすると逆効果

### ゴール
- 「通知タップ → 1タップで完了 → 終わり」
- 「通知は適切な時間帯に来てほしい」
- 「やり忘れがあっても後で通知で気づける」

### 利用シーン
- 朝7時の通知 → タップ → ワンタップ完了
- 通勤電車内で過去のログ閲覧
- 週末、まとめて記録の漏れを修正

### 重要機能
- 🟢 通知（時間帯指定／オフセット）（最重要）
- 🟢 通知ディープリンク → プリフィル済みToDo
- 🟢 ワンタップ完了
- 🟡 ホームToDo（後追い確認用）

### 不要機能
- 詳細フォーム、品種解説、SNS要素

### 行動原則（Quote）
> 「通知が来たら1タップで終わらせたい。それ以上は望まない」

## MVP / Phase 設計指針（ペルソナ起点）

### MVP（Phase 1）で必ずカバー
- **対象ペルソナ**: P1（入門者）、P2（中級者）、P7（運営）
- **理由**: 採用初期のリテンションと運用基盤を成立させる
- **必須機能**: ホーム今月のやること、月別スケジュール、品種マスタ詳細、ログ記録、お役立ち記事、Django Admin

### Phase 2 で対応
- **対象ペルソナ**: P3（多忙）、P5（記録派）、P6（通知駆動）
- **追加機能**: 期間切替＋共有、Before/After 成長比較、通知ディープリンク

### Phase 3 で対応
- **対象ペルソナ**: P4（多鉢専門家）
- **追加機能**: 一括ログ記録、品種別グルーピング、統計・分析、年間スケジュール高度化

## アンチペルソナ（対象外と明示）

| 名称 | 説明 | 対象外の理由 |
|------|-----|-----------|
| ソーシャル投稿目的のユーザー | 他ユーザーへ盆栽写真を投稿しいいねを集めたい | MVPでソーシャル機能は提供しない（レビュー対応案 §2.2） |
| 売買・商取引目的のユーザー | 盆栽の販売・購入をアプリ内で行いたい | スコープ外。専門サイトに委ねる |
| ガーデニング全般のユーザー | 盆栽以外の植物（観葉植物・野菜等）も管理したい | 盆栽特化が差別化点のためスコープ外 |
| オフライン中心の利用者 | 山林等で電波がない環境で常用したい | PWA／オフラインは Phase 2 以降（技術要件書 §8） |
